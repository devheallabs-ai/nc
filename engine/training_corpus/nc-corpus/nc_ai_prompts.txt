# NC AI — Prompt Library

These prompts drive `nc ai create`. Instead of hardcoded C templates, NC AI feeds these prompts to the built-in LLM to generate each file in a project.

## How It Works

```
User: nc ai create "a stock trading app with AI predictions"
                  |
                  v
         Parse intent & features
                  |
                  v
    Load prompt templates from this library
                  |
                  v
    Feed each prompt to nc_model_generate()
                  |
                  v
    Write generated code to project files
                  |
                  v
    Append to training corpus (self-learning)
```

## Prompt Files

| File | Purpose |
|------|---------|
| `service_backend.txt` | Generates `service.nc` — backend API with CRUD, auth, routes |
| `service_stock.txt` | Generates `service.nc` — stock market backend (NeuralEdge-style) |
| `ui_frontend.txt` | Generates `app.ncui` — frontend UI page |
| `ui_stock_dashboard.txt` | Generates `app.ncui` — stock market dashboard |
| `tests.txt` | Generates `test_app.nc` — test behaviors |

## Prompt Format

Each prompt file uses `{{variable}}` placeholders:

- `{{APP_NAME}}` — lowercase app name (e.g., `blog`, `stock`)
- `{{App_Name}}` — capitalized app name (e.g., `Blog`, `Stock`)
- `{{PROMPT}}` — original user prompt
- `{{FEATURES}}` — detected features list

## Example: Stock App Generation

```bash
nc ai create "stock market intelligence app with AI predictions portfolio management and alerts"
```

The LLM receives this prompt:

```
Generate a complete NC service for a stock market intelligence app.
The app name is "stock". Include these features: market data (quotes,
history, sectors), technical analysis (RSI, MACD, Bollinger Bands),
AI predictions (7-day forward, batch), sentiment analysis, portfolio
management (analyze, rebalance), stock screener, watchlist.
Write valid NC code with service declaration, functions, and API routes.
```

## Adding New Domains

1. Create a new prompt file in this directory
2. Add feature detection keywords in the `nc ai create` handler
3. Add prompt-code training pairs to `training_data/prompt_pairs.txt`
4. Retrain: `nc ai train`

## Self-Learning

Every project generated by `nc ai create` is appended to
`training_data/nc_corpus/user_generated.txt`. When you retrain,
the model learns from all previously generated projects.

// FILE: nc-ai/prompts/service_backend.txt

PROMPT: Generate a complete NC backend service for {{App_Name}}.
User request: "{{PROMPT}}"
App name: {{APP_NAME}}
Features: {{FEATURES}}

Write a valid NC service file with:
- service declaration and version
- Authentication (signup, login with JWT) if auth is requested
- CRUD operations: list, get, create, update, delete for {{APP_NAME}}
- Search function if search is requested
- AI analysis function if AI is requested
- health_check endpoint
- middleware (cors, log_requests, rate_limit)
- API routes mapping HTTP methods to functions

Use NC syntax:
- "service" and "version" declarations
- "to function_name with params:" for functions
- "gather items from store" for reads
- "store data into store" for writes
- "respond with data" for responses
- "api:" block with METHOD /path runs function

CODE:
// {{App_Name}} Service — generated by NC AI
// Prompt: "{{PROMPT}}"

service "{{APP_NAME}}-api"
version "1.0.0"


// FILE: nc-ai/prompts/service_stock.txt

PROMPT: Generate a complete NC stock market intelligence service like NeuralEdge.
User request: "{{PROMPT}}"
App name: {{APP_NAME}}

Write a full stock market service in NC with ALL of these sections:

1. Service declaration: service "{{APP_NAME}}-api", version "1.0.0"
2. Configure: ai_model "default", port 8000
3. Market Data:
   - get_quote(symbol): AI generates stock data JSON {symbol, price, change, change_pct, volume, market_cap, pe_ratio, week_52_high, week_52_low}
   - get_history(symbol, period): OHLCV data, handle 1w/3m/1y periods
   - get_sector_performance(): 10 US sectors with performance_pct
4. Technical Analysis:
   - analyze(symbol): RSI, MACD, SMA 20/50/200, Bollinger, ATR, support/resistance, trend, signal, confidence
   - rsi(symbol): RSI(14) with interpretation and signal
   - macd(symbol): MACD(12,26,9) with crossover and trend_strength
   - bollinger(symbol): Bands(20,2) with bandwidth, position, squeeze
5. AI Predictions:
   - predict(symbol): Run analyze first for context, 7-day predictions with key_levels, recommendation
   - predict_batch(symbols): Loop predict for each symbol
6. Sentiment:
   - sentiment(symbol): overall_sentiment, sentiment_score, news, analyst, retail, recent_news, risk_events
7. Portfolio:
   - analyze_portfolio(holdings): total_value, pnl, beta, sharpe, diversification, risk, sector_breakdown, recommendations
   - suggest_rebalance(holdings, target): Buy/sell actions with reasoning
8. Screener:
   - screen(criteria): Top 10 matching stocks with match_score
   - watchlist_summary(symbols): Loop get_quote for each
9. Health: health_check returns status "healthy"
10. Middleware: cors, log_requests, rate_limit 60
11. API routes: All endpoints under /api/v1/

Use "ask AI to" for AI-powered functions. Use "run analyze with sym" to chain analysis into predictions.

CODE:
service "{{APP_NAME}}-api"
version "1.0.0"

configure:
    ai_model is "default"
    port is 8000


// FILE: nc-ai/prompts/tests.txt

PROMPT: Generate NC test behaviors for {{App_Name}} app.
User request: "{{PROMPT}}"
App name: {{APP_NAME}}
Features: {{FEATURES}}

Write test functions in NC for:
- test_create_{{APP_NAME}}: create item, assert id not empty, assert name matches
- test_list_{{APP_NAME}}s: call list, assert returns list type
- test_health_check: call health_check, assert status is "healthy"
- test_signup (if auth): create user, assert status "created"

For stock apps, also add:
- test_get_quote: get_quote("AAPL"), assert symbol is "AAPL"
- test_analyze: analyze("AAPL"), assert symbol
- test_predict: predict("NVDA"), assert symbol
- test_sentiment: sentiment("MSFT"), assert symbol
- test_screen: screen("tech stocks with PE under 30"), assert returns list

Use NC test syntax: "to test_name:", "set result to function(args)", "assert condition, message", "respond with ok"

CODE:
// {{App_Name}} Tests — generated by NC AI
