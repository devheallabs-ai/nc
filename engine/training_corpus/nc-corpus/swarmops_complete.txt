- Proactive suggestions based on learned incident patterns

### AI Provider Support
- Works with any OpenAI-compatible API (OpenAI, Ollama, LiteLLM, Azure, etc.)
- Smart URL completion: user gives base URL, NC appends the right API path
- Only 3 settings needed: URL, key, model

### Security
- API keys masked in /config endpoint (shows "configured" not the key)
- Shell metacharacter injection protection (blocks `;|&$\`><!(){}\"\\*?[]\n\r`)
- CRLF injection protection in HTTP response headers
- Rate limiter with expired-entry eviction (prevents bypass when table full)
- JWT tokens require `exp` claim (rejects tokens without expiry)
- Secret redaction in `log` statements via `nc_redact_for_display`
- Heap overflow fix in dynamic buffer growth (cap updated only after successful realloc)
- String intern table uses tombstone deletion (prevents probe chain corruption)
- GC thread safety: mutex protection on all garbage collector operations
- Full POST body reading via Content-Length recv loop
- WebSocket connection timeout (1hr max lifetime, idle ping limit)
- Build system validates output names for shell metacharacters
- `nc doctor` shows max 4 chars of API key

### Infrastructure
- Docker: multi-stage build, Alpine 3.21, 478KB NC binary
- Kubernetes: namespace, RBAC, ConfigMap, Secret, Deployment, Service
- CI/CD: GitHub Actions for validate, build, Docker push, K8s deploy
- Makefile: `make serve` auto-copies config/.env to project root

### NC Runtime Fixes
- `write_file()` auto-creates parent directories
- `store into` works in bytecode VM (was silently discarding data)
- `nc_embed.c` API updated to current NC value types
- `coro_entry` symbol visibility fixed for macOS ARM64
- Template engine: correct JSON string context tracking for adjacent placeholders
- Auth header: API keys no longer wrapped in extra quotes
- Default AI template: standard chat completions format (not bare prompt)

### Bug Fixes
- Removed duplicate behavior definitions in server.nc (add_feedback, get_knowledge, home, ui, health)
- Removed import + inline code duplication
- Removed conflicting middleware block from MCP module
- Fixed `k8s_fix` to use `otherwise if` chain (prevents fallthrough)
- Fixed `nn_classify` heuristic priority order
- Fixed `pull_prometheus_range` to use the `duration` parameter
- Fixed `setup.nc` writing .env to wrong path
- Fixed UI: homepage loader uses `r.text()` not `r.json()`
- Fixed UI: chat response extracts clean reply text from JSON/markdown wrappers

# SwarmOps v6.0 — AI Incident Copilot

> An open-source product by **[DevHeal Labs AI](https://devheallabs.in)**

AI-powered incident investigation with self-healing, reinforcement learning, cognitive memory, and RAG.

**When your service breaks, SwarmOps automatically pulls data, finds the root cause, fixes it, learns from it, and gets smarter over time.**

Built with [NC](https://github.com/notation-as-code/nc) — single binary, zero dependencies, 478KB.

## Why SwarmOps?

### The Problem

When production breaks at 3am, engineers scramble across 5+ tools — Grafana for metrics, Kibana for logs, GitHub for deploy history, Slack for context, kubectl for K8s. They manually correlate data, guess root causes, and try fixes one by one. Every incident starts from zero.

### The Solution

SwarmOps replaces that with a single AI copilot that:

1. **Pulls data automatically** from Prometheus, Elasticsearch, GitHub, K8s
2. **Analyzes in parallel** with 4 specialized AI agents
3. **Gives a root cause** with confidence %, evidence chain, and timeline
4. **Provides exact fix commands** — copy-paste ready
5. **Learns from every incident** — gets faster and more accurate over time
6. **Self-heals** — can auto-execute fixes when confidence is high

### Why NC (Notation as Code)?

SwarmOps is built with [NC](https://github.com/notation-as-code/nc), a language designed by DevHeal Labs AI. Here's why:

| Traditional Stack | NC |
|---|---|
| Node.js/Python + 500MB dependencies | **Single 478KB binary, zero dependencies** |
| Separate web framework + ORM + AI SDK | **HTTP server, AI, JSON, file I/O all built-in** |
| Complex deployment (npm install, pip, venvs) | **Copy one binary, run** |
| Dockerfile: 20+ lines, multi-stage | **Just `nc serve service.nc`** |
| AI integration: install openai SDK, write wrapper | **`ask AI to "..." save as result`** — one line |
| Memory/caching: install Redis | **Built-in `store`, `cache`, memory functions** |

NC reads like plain English but compiles to fast bytecode. Example:

```
to investigate with service_name, description:
    ask AI to "Analyze this incident: {{description}}" save as diagnosis
    respond with diagnosis
```

No imports, no boilerplate, no package.json. The code IS the documentation.

### Where SwarmOps Has Advantage

| Compared To | SwarmOps Advantage |
|---|---|
| **PagerDuty / OpsGenie** | They alert you. SwarmOps investigates AND fixes. |
| **Datadog / New Relic** | They show dashboards. SwarmOps reads the data and tells you what's wrong. |
| **ChatGPT / Claude** | They need you to copy-paste logs. SwarmOps pulls data automatically. |
| **Custom scripts** | They don't learn. SwarmOps gets smarter with every incident (RL + memory). |
| **Rundeck / StackStorm** | They run predefined runbooks. SwarmOps generates fix commands dynamically. |

## Quick Start (3 Steps)

### Plug-and-Play Starter

The fastest way to run SwarmOps:

```bash
# 1. Clone
git clone <repo-url> && cd swarmops

# 2. Configure
cp config/.env.example config/.env
# Edit config/.env — set NC_AI_URL, NC_AI_KEY, NC_AI_MODEL

# 3. Start
./start.sh
```

Open **http://localhost:9090** — done.

`start.sh` auto-detects the NC binary, sets permissions, loads config, and starts the server. See `./start.sh help` for all commands (setup, test, docker, k8s, status, stop).

On Windows: use `start.bat` instead.

### 3 things you need

| Setting | What It Is | Example |
|---------|-----------|---------|
| `NC_AI_URL` | Your AI provider base URL | `https://api.openai.com` |
| `NC_AI_KEY` | Your API key | `sk-...` |
| `NC_AI_MODEL` | Model name | `gpt-4o-mini` |

NC auto-detects the API path — just give the base URL.

### Option 1: Docker (recommended)

```bash
docker run -p 9090:9090 \
  -e NC_AI_URL=https://api.openai.com \
  -e NC_AI_KEY=sk-your-key \
  -e NC_AI_MODEL=gpt-4o-mini \
  swarmops
```

Open **http://localhost:9090**

### Option 2: Run directly

```bash
# 1. Clone
git clone <repo-url> && cd swarmops

# 2. Configure — edit config/.env with your AI details
cp config/.env.example config/.env

# 3. Start — automatically loads config/.env
make serve
```

Open **http://localhost:9090**

### Option 3: Interactive setup

```bash
make start
# Asks for your AI URL, key, model — saves to config/.env — starts server
```

### Option 4: Build native binary

```bash
make build
./bin/swarmops serve -p 9090
```

> **Note:** You never need to manually copy `.env` files. `make serve` and `make start` automatically load `config/.env` into the project root. Just edit `config/.env` and run.

## Deploy with Kubernetes

### Quick Deploy

```bash
# 1. Build and push image
make docker
docker tag swarmops your-registry/swarmops:v6.0
docker push your-registry/swarmops:v6.0

# 2. Deploy all manifests
make k8s-deploy
```

### Step-by-Step

```bash
# 1. Create namespace
kubectl apply -f k8s/namespace.yaml

# 2. Create RBAC (ServiceAccount + ClusterRole)
kubectl apply -f k8s/rbac.yaml

# 3. Create ConfigMap
kubectl apply -f k8s/configmap.yaml

# 4. Create secrets (use your real values — never commit secrets)
kubectl create secret generic swarmops-secrets \
  --namespace swarmops \
  --from-literal=NC_AI_KEY=sk-your-key \
  --from-literal=HEAL_SECRET=your-heal-secret

# 5. Deploy
kubectl apply -f k8s/deployment.yaml

# 6. Access
kubectl port-forward -n swarmops svc/swarmops 9090:9090
```

### What's in the K8s Manifests

| File | Resources | Purpose |
|------|-----------|---------|
| `k8s/namespace.yaml` | Namespace `swarmops` | Isolated namespace |
| `k8s/rbac.yaml` | ServiceAccount + ClusterRole + Binding | Read pods/logs/events, write deploy/scale/rollback |
| `k8s/configmap.yaml` | ConfigMap `swarmops-config` | NC_AI_MODEL, NC_ALLOW_EXEC, NC_PORT, SWARMOPS_MODE |
| `k8s/secret.yaml` | Secret template | NC_AI_KEY, HEAL_SECRET, SLACK_WEBHOOK (use `kubectl create secret` instead) |
| `k8s/deployment.yaml` | Deployment + Service (ClusterIP + NodePort) | 1 replica, port 9090, health probes, resource limits |

When deployed in K8s, SwarmOps can:
- List pods across all namespaces

- Get warning/error events

## Authentication & Security

### No Auth (development)

By default, all endpoints are open. Good for local development.

### API Key Authentication (simple)

Protect all endpoints with API key(s):

```bash
# In config/.env
NC_API_KEYS=key1-for-team-a,key2-for-team-b
```

Clients pass the key:
```bash
curl -H "Authorization: Bearer key1-for-team-a" http://swarmops:9090/health
```

### JWT Authentication (enterprise)

Token-based auth with user identity:

```bash
# In config/.env
NC_JWT_SECRET=your-secret          # generate: openssl rand -base64 32
NC_JWT_USER_CLAIM=sub              # JWT claim for username
NC_JWT_ROLE_CLAIM=roles            # JWT claim for roles
```

### Self-Heal Protection

The `/self-heal` and `/rollback` endpoints execute commands — always protect them in production:

```bash
# In config/.env
HEAL_SECRET=your-heal-secret
```

### CORS (for browser access from specific domains)

```bash
# In config/.env
NC_CORS_ORIGIN=https://dashboard.company.com
```

## Technology Stack

SwarmOps is built entirely on NC (Notation as Code) — no other runtime, framework, or language is used. The entire platform, from backend to frontend to deployment, runs on a single technology.

### Platform

| Layer | Technology | Details |
|-------|-----------|---------|
| **Language** | NC (Notation as Code) | Plain-English programming language, compiles to bytecode, C11 runtime |
| **Runtime** | NC VM + JIT | Stack-based bytecode VM with computed-goto dispatch + template JIT for hot paths |
| **HTTP Server** | NC built-in | Threaded HTTP server, up to 16 concurrent workers, no nginx/Apache needed |
| **AI Engine** | NC built-in | Provider-agnostic — template-based request/response, auto-detects API path |
| **Frontend** | Single HTML file | Zero-build, no React/Vue/npm — plain HTML + CSS + JS served by NC |
| **Database** | NC in-memory store | Key-value store with optional HTTP backend, grows dynamically to 65K entries |
| **Caching** | NC built-in | Thread-safe cache with 4096 slots |
| **File I/O** | NC built-in | Auto-creates directories, sandbox-protected, path traversal prevention |
| **JSON** | NC built-in | Parser + serializer, no external library |
| **WebSocket** | NC built-in | Full WS support for real-time features |
| **Auth** | NC middleware | API key, JWT, rate limiting, CORS — all built into NC |

### AI & ML

| Component | Technology | Details |
|-----------|-----------|---------|
| **LLM Integration** | NC `ask AI to` | One-line AI calls, works with any OpenAI-compatible API |
| **Reinforcement Learning** | Custom Q-table | Q(s,a) updates with learning rate 0.1, epsilon-greedy exploration with decay |
| **Cognitive Memory** | 4-layer system | Episodic (past incidents), Semantic (patterns), Procedural (runbooks), Working (current) |
| **RAG** | NC `chunk()` + `top_k()` | Built-in text chunking and relevance ranking, no vector DB needed |
| **Neural Network** | NC `load_model()` + `predict()` | Supports sklearn (.pkl), PyTorch (.pt), TensorFlow (.h5), ONNX (.onnx) |
| **Template Engine** | NC `{{variables}}` | Handlebars-style variable interpolation in prompts and API templates |

### Infrastructure

| Component | Technology | Details |
|-----------|-----------|---------|
| **Container** | Docker (Alpine 3.21) | Multi-stage build: compile NC from C source → 478KB binary |
| **Orchestration** | Kubernetes | Namespace, RBAC, ConfigMap, Secret, Deployment, Service manifests included |
| **CI/CD** | GitHub Actions | Validate, build, Docker push, K8s deploy — triggered on push/tag |
| **Monitoring** | Prometheus + Grafana | Scrape config included, `/metrics` endpoint built-in |
| **Notifications** | Slack webhooks | Incident reports posted to Slack channels |
| **MCP** | Model Context Protocol | SwarmOps exposes tools for Claude Desktop, Cursor, and other MCP clients |

### What We Don't Use (and why)

| Not Used | Why |
|----------|-----|
| Node.js / Python / Go | NC handles everything — HTTP, AI, JSON, file I/O — in a single 478KB binary |
| React / Vue / Angular | The dashboard is a single HTML file — no build step, no npm, no webpack |
| PostgreSQL / MongoDB / Redis | NC's built-in store + file-based incident storage is sufficient; incidents are JSON files |
| Express / Flask / FastAPI | NC has a built-in threaded HTTP server with middleware |
| LangChain / LlamaIndex | NC's built-in `ask AI to`, `chunk()`, `top_k()` replace the entire RAG pipeline |
| OpenAI SDK / Anthropic SDK | NC's template-based AI engine works with any provider — zero SDKs installed |
| Docker Compose for dev | `make serve` starts everything locally — Docker is optional, for deployment only |
| Terraform / Pulumi | K8s manifests are plain YAML — `kubectl apply` is all you need |

### Binary Size Comparison

| Tool | Size | Dependencies |
|------|------|-------------|
| **SwarmOps (NC)** | **478 KB** | **Zero** |
| Typical Node.js app | 200+ MB | 500+ npm packages |
| Typical Python app | 150+ MB | 50+ pip packages |
| Typical Go app | 15-30 MB | Compiled but larger |

The entire SwarmOps platform — AI investigation, self-healing, memory, RAG, K8s monitoring, dashboard UI, MCP server — runs from a single 478KB binary with zero external dependencies.

## Integration Guide

### Prometheus

SwarmOps queries Prometheus during investigations and self-heal loops. It pulls error rates, latency p99, and CPU/memory metrics.

```bash
# config/.env
PROMETHEUS_URL=http://prometheus:9090
```

SwarmOps queries:
- `rate(http_requests_total{status=~"5..",service="<name>"}[5m])` — error rate
- `histogram_quantile(0.99, ...)` — latency p99
- `rate(container_cpu_usage_seconds_total{pod=~"<name>.*"}[5m])` — CPU

No special Prometheus configuration needed — standard PromQL is used.

### Elasticsearch

SwarmOps searches logs for errors during investigations.

```bash
# config/.env
ELASTICSEARCH_URL=http://elasticsearch:9200
```

Expected index pattern: `logs-*`. SwarmOps searches with `service:<name> AND level:ERROR`.

### Kubernetes

When deployed inside a K8s cluster, SwarmOps uses the service account to:
- List/describe pods, deployments, services
- Pull pod logs
- Get warning/error events
- Restart, rollback, scale deployments
- Delete stuck pods

The RBAC manifest (`k8s/rbac.yaml`) grants the necessary permissions.

When running locally with kubectl configured:

```bash
# No extra config needed — SwarmOps uses kubectl from PATH
```

### GitHub

SwarmOps correlates incidents with recent deployments by pulling commit history.

### Slack

Incident reports and alerts can be posted to Slack.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `nc: command not found` | NC not installed or not in PATH | Run install script: `curl -sSL https://raw.githubusercontent.com/notation-as-code/nc/main/install.sh \| bash` |
| Server starts but AI features fail | Missing or wrong AI config | Check `NC_AI_URL`, `NC_AI_KEY`, `NC_AI_MODEL` in `config/.env` |
| `Permission denied` on start.sh | File not executable | Run `chmod +x start.sh` |
| Port 9090 already in use | Another process on that port | Set `NC_PORT=9091` in `config/.env`, or stop the other process |
| K8s features return errors | kubectl not configured | Install kubectl and configure cluster access |
| RAG search returns 0 results | No docs indexed | Put `.md` or `.txt` files in the `docs/` directory |
| Self-heal returns 401 | Missing or wrong HEAL_SECRET | Pass matching `secret` parameter in request |
| Docker build fails | NC source not found | Ensure NC source is at `../notation as code/nc/` or adjust Makefile path |
| Tests fail with connection refused | Server not running | Start the server first: `./start.sh` |
| AI returns garbled output | Model doesn't follow JSON instructions | Try a stronger model (gpt-4o) or check prompt in server logs |

| Version | Supported |
|---------|-----------|
| v6.0.x  | Yes       |
| < v6.0  | No        |

## Security Features

SwarmOps includes the following security measures:

### Authentication
- **API Key auth** — `NC_API_KEYS` env var, validated per request
- **JWT auth** — `NC_JWT_SECRET`, tokens require `exp` claim
- **Self-heal protection** — `HEAL_SECRET` required for `/self-heal` and `/rollback`

### Input Validation
- Shell metacharacter injection protection (blocks `;|&$\`><!(){}\"\\*?[]\n\r`)
- HTTP header CRLF injection prevention
- Path traversal protection with `realpath()` verification
- Query parameter null-byte handling

### Secret Management
- API keys masked in `/config` endpoint (shows `"configured"`, not the key)
- Secrets redacted in `log` statements via `nc_redact_for_display`
- `nc doctor` shows max 4 characters of API keys
- `.env` files git-ignored by default

### Rate Limiting
- Per-IP rate limiting with configurable window
- Rate limiter table with expired-entry eviction (prevents bypass)
- WebSocket connection timeout (1hr max lifetime)

### Runtime Protection
- GC thread safety with mutex protection
- Heap overflow prevention in dynamic buffers
- Content-Length-aware request body reading
- Build system validates output names for injection

## Best Practices for Deployment

1. **Always set `HEAL_SECRET`** — never leave self-heal endpoints unprotected
2. **Use `NC_API_KEYS` or `NC_JWT_SECRET`** in production — don't run without auth
3. **Set `NC_CORS_ORIGIN`** — don't allow all origins in production
4. **Never commit `.env` files** — they're git-ignored by default
5. **Use K8s Secrets** for sensitive values — don't put keys in ConfigMaps
6. **Keep NC updated** — security patches are applied to the runtime

## Disclosure Policy

- We follow responsible disclosure
- Security fixes are released as patch versions
- Critical vulnerabilities are patched within 7 days
- Advisory published after fix is available

## How Agents Work

SwarmOps uses specialized AI agents that run during an investigation. Each agent has a focused role and receives the same collected data. The server orchestrates them from `src/agents/investigate.nc`.

### Agent Pipeline

```
Incident Reported
       │
       ▼
┌─ Data Pull Agent (data-pull.nc) ─────────────────────┐
│  Queries Prometheus, Elasticsearch, GitHub, health    │
│  Returns: metrics_data, log_data, deploy_data, health │
└──────────────────────────────────────────────────────┘
       │
       ▼
┌─ 4 AI Agents (investigate.nc) ── run in parallel ────┐
│  1. Telemetry Agent  — analyzes error patterns        │
│  2. Deploy Agent     — correlates recent changes      │
│  3. Hypothesis Agent — determines root cause          │
│  4. Remediation Agent — generates fix commands        │
└──────────────────────────────────────────────────────┘
       │
       ▼
┌─ Memory + Learning ─────────────────────────────────┐
│  Saves to episodic memory, updates RL Q-table        │
│  Extracts semantic patterns on feedback               │
└──────────────────────────────────────────────────────┘
```

### How an Agent is Defined

In NC, an agent is simply a `to` behavior (function) that:
1. Receives context data (metrics, logs, deploys, memory)
2. Calls `ask AI to "..."` with a structured prompt
3. Returns JSON with its analysis

Example from `investigate.nc`:

```nc
ask AI to "You are the Telemetry Agent. Analyze these metrics and logs for {{service_name}}.
  METRICS: {{metrics_data}}
  LOGS: {{log_data}}
  Return ONLY valid JSON: {\"error_pattern\": \"string\", \"severity\": \"string\"}"
  save as telemetry_analysis
```

The key pattern: NC's `ask AI to` with `{{variable}}` interpolation handles prompt templating, API calls, and JSON parsing in one line.

## How RL Q-Learning Works

SwarmOps uses tabular Q-learning to improve investigation strategy selection over time.

### Q-Table Structure

Stored in `knowledge/rl-policy.json`:

```json
{
  "states": { ... },
  "total_episodes": 42,
  "alpha": 0.1,
  "epsilon": 0.15
}
```

- **State**: incident category (e.g., `"deployment"`, `"resource-exhaustion"`, `"network"`)
- **Action**: investigation strategy that was used
- **Reward**: `+1.0` if feedback says correct, `-0.5` if wrong, `0` if no feedback

### Update Rule

```
Q(s,a) = Q(s,a) + alpha * (reward - Q(s,a))
```

- `alpha = 0.1` (learning rate)
- `epsilon` starts at `0.2` and decays by `0.95` per episode (explore less over time)
- Minimum epsilon: `0.05`

### Feedback Loop

1. Engineer investigates with SwarmOps
2. SwarmOps returns diagnosis with category and root cause
3. Engineer marks diagnosis as correct or wrong via `/feedback` endpoint
4. `add_feedback` in `cognitive.nc` updates the Q-table
5. If correct: also updates semantic memory (generalized pattern) and procedural memory (runbook)
6. Next investigation uses updated Q-values to select strategy

### When It Activates

The RL policy is loaded at the start of every investigation in `investigate.nc`. If the Q-table has data for the current state (incident category), it biases the investigation strategy. With epsilon-greedy exploration, it occasionally tries new strategies.

## How RAG Knowledge Base Works

RAG (Retrieval Augmented Generation) lets SwarmOps use your docs as context during AI analysis.

### Indexing

1. Place `.md`, `.txt`, or `.nc` files in the `docs/` directory
2. Call `GET /rag/index` or let it index automatically during investigation
3. NC's built-in `chunk()` splits docs into 800-token chunks with 100-token overlap
4. Chunks are stored in memory for the session

### Retrieval

1. During investigation, `top_k(doc_chunks, 3)` finds the 3 most relevant chunks
2. Relevance is determined by keyword matching (NC built-in)
3. Retrieved chunks are injected into the AI prompt as `RELEVANT DOCS (RAG):`

### Search API

```
POST /rag/search
{"query": "CrashLoopBackOff fix"}
```

Returns matching chunks from your docs, ranked by relevance.

### What to Put in docs/

- Runbooks (e.g., `runbook-checkout.md`)
- Architecture documentation
- On-call guides
- Known issue patterns
- SRE best practices

The more relevant docs you add, the better SwarmOps diagnoses issues specific to your infrastructure.

## How to Add a New Agent

### Step 1: Create the Agent File

Create `src/agents/your-agent.nc`:

```nc
// ══════════════════════════════════════════════════════════════════
//  YOUR AGENT — Description of what it does
// ══════════════════════════════════════════════════════════════════

to your_agent_analyze with service_name, context_data:
    purpose: "Describe what this agent does"

    ask AI to "You are the YourAgent for SwarmOps. Analyze: {{context_data}}.
    Service: {{service_name}}.
    Return ONLY valid JSON: {\"finding\": \"string\", \"confidence\": 0, \"recommendation\": \"string\"}"
    save as result

    respond with result
```

### Step 2: Wire It Into the Server

In `src/server.nc`, add a route that calls your agent:

```nc
on POST "/your-agent/analyze":
    set service to body.service_name
    set context to body.context
    call your_agent_analyze with service, context save as result
    respond with result
```

### Step 3: Include in Investigation Pipeline (Optional)

If your agent should run during every investigation, add it to the parallel agent section in `src/agents/investigate.nc`:

```nc
// Add alongside existing agents
ask AI to "You are YourAgent. ..." save as your_analysis
```

### Step 4: Add Tests

Add test cases to `tests/test-api.nc`:

```nc
gather r from base + "/your-agent/analyze":
    method: "POST"
    body: {"service_name": "test-svc", "context": "test data"}
if r.finding:
    show "  PASS  /your-agent/analyze"
    set pass to pass + 1
```

### Step 5: Register in MCP (Optional)

If the agent should be available via MCP (Claude Desktop, Cursor), add it to `src/mcp/tools.nc`.

## How to Add a New Integration

### Data Source Integration

SwarmOps integrations pull data from external services. They live in `src/agents/data-pull.nc`.

#### Step 1: Add Environment Variable

In `config/.env.example` and `.env.example`:

```bash
# Your Integration
# YOUR_SERVICE_URL=http://your-service:port
# YOUR_SERVICE_TOKEN=your-token
```

#### Step 2: Create the Pull Function

In `src/agents/data-pull.nc`:

```nc
to pull_your_service with query:
    purpose: "Pull data from YourService"
    set service_url to env("YOUR_SERVICE_URL")
    if service_url:
        set url to service_url + "/api/search?q=" + query
        gather result from url:
            headers: {"Authorization": "Bearer " + env("YOUR_SERVICE_TOKEN")}
        respond with {"source": "your_service", "data": result}
    otherwise:
        respond with {"source": "your_service", "error": "YOUR_SERVICE_URL not configured"}
```

#### Step 3: Add a Route

In `src/server.nc`:

```nc
on POST "/pull/your-service":
    call pull_your_service with body.query save as result
    respond with result
```

#### Step 4: Wire Into Investigation

In `src/agents/investigate.nc`, add a data pull step alongside Prometheus/ES/GitHub:

```nc
set your_data to "No YourService configured"
set your_url to env("YOUR_SERVICE_URL")
if your_url:
    call pull_your_service with service_name save as your_data
```

Then include `your_data` in the AI context string.

### Notification Integration

For outbound notifications (like Slack), add to the self-heal or investigation response:

```nc
to notify_your_service with message:
    set webhook to env("YOUR_WEBHOOK_URL")
    if webhook:
        gather response from webhook:
            method: "POST"
            body: {"text": message}
```

### Prerequisites

The SwarmOps server must be running before tests:

./start.sh

./start.sh test

### Test Files

| File | What It Tests | Entry Point |

| `test-issues.nc` | NC runtime regression tests (run as server on port 9877) | N/A |
| `test-runner.nc` | Runs NC runtime regression tests | `run_tests` |

### Running Individual Tests

```bash
# API tests only
NC_ALLOW_EXEC=1 nc run tests/test-api.nc -b test_all

# Memory tests only
NC_ALLOW_EXEC=1 nc run tests/test-memory.nc -b test_memory

# Integration tests only
NC_ALLOW_EXEC=1 nc run tests/test_integrations.nc -b test_integrations
```

### Writing Tests

Tests in NC follow this pattern:

```nc
to test_something:
    set pass to 0
    set fail to 0

    gather result from "http://localhost:9090/endpoint":
        method: "POST"
        body: {"key": "value"}

    if result.expected_field:
        show "  PASS  description"
        set pass to pass + 1
    otherwise:
        show "  FAIL  description"
        set fail to fail + 1

    show "Results: " + str(pass) + " passed, " + str(fail) + " failed"
```

Key patterns:
- Use `gather ... from` for HTTP requests
- Use `if ... otherwise` for assertions
- Count pass/fail and show summary
- Test graceful error handling (unconfigured services should not crash)

### Validation

Check NC syntax without running:

```bash
make validate
```

- NC files use `//` for comments

# Incident Patterns

Common incident patterns that SwarmOps learns to recognize over time.

## Deployment-Related

### Credential Misconfiguration
- **Signals**: Auth errors spike immediately after deploy
- **Pattern**: `deploy + auth_error = credential_misconfiguration`
- **Root cause**: New deployment changed secrets/env vars or migrated credential storage
- **Fix**: Check secrets, compare old vs new config, rollback if needed

### Bad Code Deploy
- **Signals**: Error rate spikes, new error messages, recent deploy within 30 min
- **Pattern**: `deploy + error_spike + new_error_messages = bad_deploy`
- **Root cause**: Code bug introduced in the latest deployment
- **Fix**: `kubectl rollout undo deployment/<name>`

### Config Change
- **Signals**: Behavior change without code deploy, config-related errors
- **Pattern**: `config_change + service_behavior_change = config_error`
- **Root cause**: ConfigMap or environment variable changed
- **Fix**: Check recent ConfigMap/Secret changes, compare with working version

## Infrastructure

### Resource Exhaustion
- **Signals**: OOMKilled, high CPU, connection pool exhausted, disk full
- **Pattern**: `high_resource_usage + pod_restarts = resource_exhaustion`
- **Root cause**: Memory/CPU limits too low, or memory leak in application
- **Fix**: Increase resource limits, check for memory leaks

### Network Partition
- **Signals**: Timeout errors, connection refused, DNS failures
- **Pattern**: `timeouts + connection_refused + healthy_target = network_issue`
- **Root cause**: Network policy, security group, or DNS misconfiguration
- **Fix**: Check network policies, verify DNS, check security groups

### Database Issues
- **Signals**: Connection refused, slow queries, connection pool exhausted
- **Pattern**: `db_connection_error + high_latency = database_issue`
- **Root cause**: Database overloaded, credentials rotated, connection pool exhausted
- **Fix**: Check DB health, restart connection pools, verify credentials

## Cascading Failures

### Upstream Dependency Down
- **Signals**: Timeouts to specific upstream, error rate correlated with upstream
- **Pattern**: `upstream_timeout + upstream_error_rate = dependency_failure`
- **Root cause**: An upstream service is failing, causing cascading errors
- **Fix**: Enable circuit breakers, check upstream service health

### Retry Storm
- **Signals**: Exponential load increase, all instances saturated
- **Pattern**: `load_spike + retry_patterns + no_deploy = retry_storm`
- **Root cause**: Failed request retries overwhelming the service
- **Fix**: Add backoff to retries, enable circuit breakers, shed load

# Kubernetes Troubleshooting

Quick reference for K8s issues that SwarmOps can detect and fix.

## Pod Issues

### CrashLoopBackOff
```bash
kubectl describe pod <name> -n <ns>      # Check events
kubectl logs <name> -n <ns> --previous   # Check crash logs
```
- **Common causes**: missing config, wrong image, failing health check
- **SwarmOps**: POST /k8s/fix `{"action":"restart","target":"<deploy>","namespace":"<ns>"}`

### OOMKilled
```bash
kubectl top pods -n <ns>                  # Check resource usage
kubectl describe pod <name> -n <ns>      # Check limits vs actual
```
- **Fix**: increase memory limits in deployment spec
- **SwarmOps**: POST /k8s/investigate `{"namespace":"<ns>","description":"OOM killed"}`

### ImagePullBackOff
```bash
kubectl describe pod <name> -n <ns>      # Check image name + registry auth
```
- **Common causes**: wrong image tag, private registry without imagePullSecret

### Pending (unschedulable)
```bash
kubectl describe pod <name> -n <ns>      # Check scheduling events
kubectl get nodes                         # Check node capacity
```
- **Common causes**: insufficient resources, node selector mismatch, taints

## Deployment Issues

### Rollback
```bash
kubectl rollout undo deployment/<name> -n <ns>
kubectl rollout status deployment/<name> -n <ns>
```
- **SwarmOps**: POST /k8s/fix `{"action":"rollback","target":"<deploy>","namespace":"<ns>"}`

### Scale Up
```bash
kubectl scale deployment/<name> --replicas=5 -n <ns>
```
- **SwarmOps**: POST /k8s/fix `{"action":"scale","target":"<deploy>","namespace":"<ns>"}`

### Restart (rolling)
```bash
kubectl rollout restart deployment/<name> -n <ns>
```
- **SwarmOps**: POST /k8s/fix `{"action":"restart","target":"<deploy>","namespace":"<ns>"}`

## Common Incident Categories

| Category | Typical Signals | Common Fix |
|----------|----------------|------------|
| `deployment` | Recent deploy + errors spike | Rollback: `kubectl rollout undo` |
| `configuration` | Auth errors after deploy | Check secrets/configmaps |
| `infrastructure` | High latency + CPU/memory pressure | Scale up or restart |
| `code-bug` | Error rate spike, specific error messages | Hotfix or rollback |
| `dependency` | Upstream timeouts, connection refused | Check dependent services |
| `resource-exhaustion` | OOM killed, disk full, connection pool exhausted | Increase limits |

- If SwarmOps confidence < 50%: escalate to senior engineer
- If self-heal fails verification: manual investigation required

Guidelines for running SwarmOps in production.

## Incident Response

### Severity Levels
| Level | Criteria | Response Time |
|-------|----------|---------------|
| Critical | Service down, data loss risk | Immediate |
| High | Major feature broken, > 50% error rate | 15 min |
| Medium | Degraded performance, partial outage | 1 hour |
| Low | Minor issue, workaround available | Next business day |

### SwarmOps Confidence Guide
| Confidence | Action |
|------------|--------|
| > 85% | Safe to auto-fix with self-heal |
| 70-85% | Review fix commands before executing |
| 50-70% | Use as starting point, investigate further |
| < 50% | Manual investigation needed |

## Self-Heal Safety

### Enable Auto-Fix
```bash
HEAL_SECRET=your-secret-here   # Required for /self-heal and /rollback
```

### Safety Rules (built into SwarmOps)
- Only executes fixes when confidence > 70%
- Skips high-risk commands automatically
- Verifies fix worked after execution
- Saves rollback commands for every fix
- Cooldown period between heal attempts
- All actions logged to incidents/ folder

## Learning Loop

SwarmOps gets smarter with every incident:

1. **Investigate** → creates incident report
2. **Feedback** → engineer says if diagnosis was correct
3. **RL Update** → Q-table adjusts strategy weights
4. **Semantic Memory** → generalized pattern extracted
5. **Procedural Memory** → fix sequence saved as runbook
6. **Next Incident** → patterns and runbooks used as context

        set range_seconds to duration
    otherwise:
        set range_seconds to 3600
    if prom_url:
        set full_url to prom_url + "/api/v1/query_range?query=" + query + "&start=" + str(time_now() - range_seconds) + "&end=" + str(time_now()) + "&step=60"
        gather result from full_url
        respond with {"source": "prometheus_range", "query": query, "data": result}
    otherwise:
        respond with {"source": "prometheus_range", "error": "PROMETHEUS_URL not configured"}

    set es_url to env("ELASTICSEARCH_URL")

        respond with {"source": "elasticsearch", "query": search_query, "data": result}

        respond with {"source": "elasticsearch", "error": "ELASTICSEARCH_URL not configured"}

        set deploy_url to "https://api.github.com/repos/" + gh_repo + "/deployments?per_page=10"

            headers: {"Authorization": "Bearer " + gh_token, "Accept": "application/vnd.github.v3+json"}

// ══════════════════════════════════════════════════════════════════
//  INVESTIGATE — The core: gather real data, AI analyzes it
//  With: MEMORY + RL + RAG + optional Neural Network
// ══════════════════════════════════════════════════════════════════

to investigate with service_name, description:
    purpose: "Full automated investigation with learning from past incidents"
    log "INVESTIGATION STARTED: {{service_name}} — {{description}}"

    // Load cognitive memory — 4 layers + RAG
    shell("mkdir -p incidents knowledge docs")

    // Episodic: chunk and retrieve relevant past incidents
    set all_incidents to shell("ls incidents/*.json 2>/dev/null | tail -15 | while read f; do cat \"$f\" 2>/dev/null | head -c 300; echo '---'; done || echo NONE")
    set past_raw to all_incidents
    if all_incidents is not equal "NONE":
        set incident_chunks to chunk(all_incidents, 400, 50)
        set past_raw to str(top_k(incident_chunks, 5))

    // Semantic + Procedural + RL
    set semantic_mem to "none"
    if file_exists("knowledge/semantic.json"):
        set semantic_mem to read_file("knowledge/semantic.json")
    set procedural_mem to "none"
    if file_exists("knowledge/procedural.json"):
        set procedural_mem to read_file("knowledge/procedural.json")
    set rl_policy to "none"
    if file_exists("knowledge/rl-policy.json"):
        set rl_policy to read_file("knowledge/rl-policy.json")

    // RAG: load relevant docs (runbooks, architecture docs)
    set rag_context to "none"
    set all_docs to shell("cat docs/*.md docs/*.txt 2>/dev/null || echo ''")
    if len(all_docs) is above 10:
        set doc_chunks to chunk(all_docs, 600, 80)
        set rag_context to str(top_k(doc_chunks, 3))
        log "RAG: loaded " + str(len(doc_chunks)) + " doc chunks"

    set knowledge_raw to "SEMANTIC MEMORY: " + str(semantic_mem) + ". PROCEDURAL MEMORY: " + str(procedural_mem) + ". RL POLICY: " + str(rl_policy) + ". RELEVANT DOCS (RAG): " + str(rag_context)
    log "Loaded: episodic + semantic + procedural + RL + RAG"

    set prom_url to env("PROMETHEUS_URL")
    set es_url to env("ELASTICSEARCH_URL")
    set gh_token to env("GITHUB_TOKEN")

    set metrics_data to "No Prometheus configured"
    set log_data to "No Elasticsearch configured"
    set deploy_data to "No GitHub configured"
    set health_data to "No health endpoints configured"

    if prom_url:
        set error_query to prom_url + "/api/v1/query?query=rate(http_requests_total{status=~\"5..\",service=\"" + service_name + "\"}[5m])"
        gather prom_errors from error_query
        set latency_query to prom_url + "/api/v1/query?query=histogram_quantile(0.99,rate(http_request_duration_seconds_bucket{service=\"" + service_name + "\"}[5m]))"
        gather prom_latency from latency_query
        set cpu_query to prom_url + "/api/v1/query?query=rate(container_cpu_usage_seconds_total{pod=~\"" + service_name + ".*\"}[5m])"
        gather prom_cpu from cpu_query
        set metrics_data to json_encode({"error_rate": prom_errors, "latency_p99": prom_latency, "cpu": prom_cpu})
        log "Pulled Prometheus metrics for {{service_name}}"

    if es_url:
        set log_url to es_url + "/logs-*/_search?q=service:" + service_name + "+AND+level:ERROR&size=30&sort=@timestamp:desc"
        gather es_logs from log_url
        set log_data to json_encode(es_logs)
        log "Pulled Elasticsearch logs for {{service_name}}"

    if gh_token:
        set gh_repo to env("GITHUB_REPO")
        set deploy_url to "https://api.github.com/repos/" + gh_repo + "/commits?per_page=10"
        gather gh_commits from deploy_url:
            headers: {"Authorization": "Bearer " + gh_token, "Accept": "application/vnd.github.v3+json"}
        set deploy_data to json_encode(gh_commits)
        log "Pulled GitHub deploy history"

    set urls_str to env("MONITOR_URLS")
    if urls_str:
        set health_data to shell("for url in $(echo '" + urls_str + "' | tr ',' ' '); do echo \"$url: $(curl -s -o /dev/null -w '%{http_code}' --connect-timeout 3 $url 2>/dev/null)\"; done")
        log "Checked health endpoints"

    set full_context to "SERVICE: {{service_name}}. ISSUE: {{description}}. METRICS FROM PROMETHEUS: {{metrics_data}}. LOGS FROM ELASTICSEARCH: {{log_data}}. DEPLOYS FROM GITHUB: {{deploy_data}}. HEALTH CHECKS: {{health_data}}."

    ask AI to "You are the Log and Metrics Analysis Agent. Analyze the real telemetry data from this production incident. Identify error patterns in logs, metric anomalies (latency spikes, error rate increases, CPU/memory pressure). Context: {{full_context}}. Return ONLY valid JSON: {\"agent\": \"telemetry-analysis\", \"signals\": [{\"name\": \"string\", \"severity\": \"critical|high|medium|low\", \"evidence\": \"string (quote specific values)\"}], \"error_pattern\": \"string\", \"metric_anomalies\": [{\"metric\": \"string\", \"value\": \"string\", \"normal\": \"string\"}], \"summary\": \"string\"}" save as telemetry_agent

    set tel_valid to validate(telemetry_agent, ["signals", "summary"])
    if tel_valid.valid is not equal yes:
        log "Telemetry Agent returned invalid response"
        set telemetry_agent to {"agent": "telemetry-analysis", "signals": [], "error_pattern": "AI response invalid", "metric_anomalies": [], "summary": "Analysis failed — AI returned unexpected format"}
    otherwise:
        log "Telemetry Agent: " + telemetry_agent.summary

    ask AI to "You are the Deployment and Dependency Agent. Check if recent code changes or deployments correlate with the incident. Also determine if failures cascade between services. Context: {{full_context}}. Return ONLY valid JSON: {\"agent\": \"deploy-dependency\", \"signals\": [{\"name\": \"string\", \"severity\": \"critical|high|medium|low\", \"evidence\": \"string\"}], \"suspicious_changes\": [{\"what\": \"string\", \"when\": \"string\", \"correlation\": \"string\"}], \"dependency_chain\": [{\"from\": \"string\", \"to\": \"string\", \"impact\": \"string\"}], \"is_deploy_related\": true, \"summary\": \"string\"}" save as deploy_dep_agent

    set dep_valid to validate(deploy_dep_agent, ["signals", "summary"])
    if dep_valid.valid is not equal yes:
        log "Deploy/Dep Agent returned invalid response"
        set deploy_dep_agent to {"agent": "deploy-dependency", "signals": [], "suspicious_changes": [], "dependency_chain": [], "is_deploy_related": false, "summary": "Analysis failed — AI returned unexpected format"}
    otherwise:
        log "Deploy/Dep Agent: " + deploy_dep_agent.summary

    set all_evidence to json_encode({"telemetry": telemetry_agent, "deploy_dependency": deploy_dep_agent})

    ask AI to "You are the Hypothesis Agent with COGNITIVE MEMORY and REINFORCEMENT LEARNING. You have 4 memory layers: EPISODIC (past incidents): {{past_raw}}. SEMANTIC (generalized patterns): {{semantic_mem}}. PROCEDURAL (learned fix sequences): {{procedural_mem}}. RL POLICY (what strategies work): {{rl_policy}}. CURRENT EVIDENCE: {{all_evidence}}. RULES: 1) If this matches a pattern in semantic memory, boost confidence by 15%. 2) If procedural memory has a runbook for this pattern, recommend that fix first. 3) If RL policy shows a strategy worked for this category, note it. 4) If this is a NEW pattern not in any memory, flag it for learning. Return ONLY valid JSON: {\"root_cause\": \"string\", \"confidence_percent\": 0, \"category\": \"configuration|deployment|code-bug|infrastructure|dependency|resource-exhaustion|network|security\", \"evidence_chain\": [\"string\"], \"timeline\": [{\"time\": \"string\", \"event\": \"string\"}], \"affected_services\": [\"string\"], \"memory_used\": {\"episodic_match\": \"string (past incident ID or 'none')\", \"semantic_match\": \"string (pattern matched or 'new')\", \"procedural_match\": \"string (runbook used or 'none')\", \"rl_strategy\": \"string (strategy from RL or 'explore')\"}, \"learning\": \"string (what to add to semantic memory from this incident)\"}" save as hypothesis

    set hyp_valid to validate(hypothesis, ["root_cause", "confidence_percent"])
    if hyp_valid.valid is not equal yes:
        set hypothesis to {"root_cause": "Unable to determine — AI response invalid", "confidence_percent": 0, "category": "unknown", "evidence_chain": [], "timeline": [], "affected_services": [service_name]}
    otherwise:
        log "Root Cause: " + hypothesis.root_cause + " (" + str(hypothesis.confidence_percent) + "%)"

    set diagnosis to json_encode(hypothesis)
    ask AI to "You are the Remediation Agent. Provide specific fix actions with exact CLI commands. The engineer should be able to copy-paste these commands to fix the issue. Diagnosis: {{diagnosis}}. Return ONLY valid JSON: {\"immediate_actions\": [{\"step\": 0, \"action\": \"string\", \"command\": \"string\", \"risk\": \"low|medium|high\"}], \"verification\": [\"string\"], \"prevention\": [\"string\"], \"estimated_recovery_minutes\": 0, \"rollback\": \"string\"}" save as remediation

    set rem_valid to validate(remediation, ["immediate_actions"])
    if rem_valid.valid is not equal yes:
        set remediation to {"immediate_actions": [{"step": 1, "action": "Manual investigation required — AI could not generate fix commands", "command": "kubectl get pods -A | grep -v Running", "risk": "low"}], "verification": ["Check service health endpoint"], "prevention": ["Add monitoring alerts"], "estimated_recovery_minutes": 30, "rollback": "No automated rollback available"}

    set incident_id to "INC-" + str(floor(random() * 9000 + 1000))
    set signal_count to len(telemetry_agent.signals) + len(deploy_dep_agent.signals)

    set report to {"incident_id": incident_id, "service": service_name, "description": description, "root_cause": hypothesis.root_cause, "confidence": hypothesis.confidence_percent, "category": hypothesis.category, "affected_services": hypothesis.affected_services, "evidence_chain": hypothesis.evidence_chain, "timeline": hypothesis.timeline, "agents": {"telemetry": telemetry_agent, "deploy_dependency": deploy_dep_agent}, "remediation": remediation, "signals_total": signal_count, "data_sources": {"prometheus": prom_url, "elasticsearch": es_url, "github": gh_token}, "investigated_at": time_now()}

    shell("mkdir -p incidents")
    write_file("incidents/" + incident_id + ".json", json_encode(report))
    log "Saved: incidents/" + incident_id + ".json"

    set slack_url to env("SLACK_WEBHOOK")
    if slack_url:
        set slack_msg to "{\"text\":\"*SwarmOps Incident Report — " + incident_id + "*\\n*Service:* " + service_name + "\\n*Root Cause:* " + hypothesis.root_cause + "\\n*Confidence:* " + str(hypothesis.confidence_percent) + "%\\n*Category:* " + hypothesis.category + "\\n*Fix:* " + remediation.immediate_actions[0].action + "\"}"
        gather slack_result from slack_url:
            method: "POST"
            body: slack_msg
        log "Posted to Slack"

    respond with report

// === FILE: swarmops/src/agents/self-heal.nc ===
// ══════════════════════════════════════════════════════════════════
//  SELF-HEAL — Detect → Diagnose → Fix → Verify → Rollback
// ══════════════════════════════════════════════════════════════════

to self_heal with service_name, description, auto_execute, secret:
    purpose: "Full self-healing loop: detect, diagnose, fix, verify"
    set expected to env("HEAL_SECRET")
    if expected:
        if secret is not equal expected:
            respond with {"error": "Unauthorized. Pass secret parameter matching HEAL_SECRET env var.", "_status": 401}

    // Cooldown: prevent re-heal within 60 seconds
    if file_exists(".last_heal"):
        set last_heal_time to read_file(".last_heal")

    write_file(".last_heal", str(time_now()))
    log "SELF-HEAL STARTED: {{service_name}}"

    set heal_log to "=== Self-Heal Report ===\nService: " + service_name + "\nStarted: " + str(time_now()) + "\n\n"

    // Step 1: Gather current state
    set heal_log to heal_log + "STEP 1: Gathering system state...\n"
    set health_data to "unknown"
    set urls_str to env("MONITOR_URLS")
    if urls_str:
        set health_data to shell("for url in $(echo '" + urls_str + "' | tr ',' ' '); do echo \"$url: $(curl -s -o /dev/null -w '%{http_code}' --connect-timeout 3 $url 2>/dev/null)\"; done")

    set metrics_data to "not configured"
    set prom_url to env("PROMETHEUS_URL")
    if prom_url:
        set error_q to prom_url + "/api/v1/query?query=rate(http_requests_total{status=~\"5..\",service=\"" + service_name + "\"}[5m])"
        gather metrics_data from error_q

    set log_data to "not configured"
    set es_url to env("ELASTICSEARCH_URL")
    if es_url:
        set log_url to es_url + "/logs-*/_search?q=service:" + service_name + "+AND+level:ERROR&size=20&sort=@timestamp:desc"
        gather log_data from log_url

    set deploy_data to "not configured"
    set gh_token to env("GITHUB_TOKEN")
    if gh_token:
        set gh_repo to env("GITHUB_REPO")
        set deploy_url to "https://api.github.com/repos/" + gh_repo + "/commits?per_page=5"
        gather deploy_data from deploy_url:
            headers: {"Authorization": "Bearer " + gh_token}

    set heal_log to heal_log + "  Health: " + str(health_data) + "\n  Metrics: collected\n  Logs: collected\n  Deploys: collected\n\n"

    // Step 2: AI Diagnosis
    set heal_log to heal_log + "STEP 2: AI diagnosing root cause...\n"
    set full_context to "SERVICE: {{service_name}}. ISSUE: {{description}}. HEALTH: " + str(health_data) + ". METRICS: " + str(metrics_data) + ". LOGS: " + str(log_data) + ". DEPLOYS: " + str(deploy_data) + "."

    ask AI to "You are SwarmOps Self-Healing Agent. Diagnose this incident AND provide exact fix commands that can be executed automatically. IMPORTANT: Commands must be safe, idempotent, and reversible. Context: {{full_context}}. Return ONLY valid JSON: {\"root_cause\": \"string\", \"confidence_percent\": 0, \"category\": \"string\", \"fix_commands\": [{\"command\": \"string (exact shell command)\", \"description\": \"string\", \"risk\": \"low|medium|high\", \"reversible\": true}], \"verify_command\": \"string (command to check if fix worked)\", \"rollback_commands\": [{\"command\": \"string\", \"description\": \"string\"}], \"safe_to_auto_execute\": true}" save as diagnosis

    set heal_log to heal_log + "  Root cause: " + diagnosis.root_cause + "\n  Confidence: " + str(diagnosis.confidence_percent) + "%\n  Category: " + diagnosis.category + "\n\n"

    // Step 3: Execute fix (only if auto_execute is "yes" and confidence > 70)
    set executed to "no"
    set exec_results to []
    if auto_execute is equal "yes":
        if diagnosis.confidence_percent is above 70:
            if diagnosis.safe_to_auto_execute:
                set heal_log to heal_log + "STEP 3: AUTO-EXECUTING FIX (confidence > 70%, safe=true)...\n"
                repeat for each fix_cmd in diagnosis.fix_commands:
                    if fix_cmd.risk is not equal "high":
                        log "EXECUTING: " + fix_cmd.command
                        set cmd_result to shell(fix_cmd.command)
                        set exec_results to exec_results + [{"command": fix_cmd.command, "result": cmd_result}]
                        set heal_log to heal_log + "  Executed: " + fix_cmd.command + "\n  Result: " + str(cmd_result) + "\n"
                    otherwise:
                        set heal_log to heal_log + "  SKIPPED (high risk): " + fix_cmd.command + "\n"
                        set exec_results to exec_results + [{"command": fix_cmd.command, "result": "SKIPPED — high risk"}]
                set executed to "yes"
            otherwise:
                set heal_log to heal_log + "STEP 3: SKIPPED — AI marked as unsafe for auto-execution\n\n"
        otherwise:
            set heal_log to heal_log + "STEP 3: SKIPPED — confidence " + str(diagnosis.confidence_percent) + "% below 70% threshold\n\n"
    otherwise:
        set heal_log to heal_log + "STEP 3: SKIPPED — auto_execute not enabled (set to 'yes' to enable)\n\n"

    // Step 4: Verify fix worked
    set verification to "not run"
    if executed is equal "yes":
        set heal_log to heal_log + "\nSTEP 4: VERIFYING FIX...\n"
        if diagnosis.verify_command:
            set verification to shell(diagnosis.verify_command)
            set heal_log to heal_log + "  Command: " + diagnosis.verify_command + "\n  Result: " + str(verification) + "\n"

        // Step 5: Re-check health after fix
        set heal_log to heal_log + "\nSTEP 5: RE-CHECKING HEALTH...\n"
        if urls_str:
            set post_health to shell("for url in $(echo '" + urls_str + "' | tr ',' ' '); do echo \"$url: $(curl -s -o /dev/null -w '%{http_code}' --connect-timeout 3 $url 2>/dev/null)\"; done")
            set heal_log to heal_log + "  Post-fix health: " + str(post_health) + "\n"

    // Save report
    set incident_id to "HEAL-" + str(floor(random() * 9000 + 1000))
    set report to {"incident_id": incident_id, "service": service_name, "description": description, "root_cause": diagnosis.root_cause, "confidence": diagnosis.confidence_percent, "category": diagnosis.category, "fix_commands": diagnosis.fix_commands, "rollback_commands": diagnosis.rollback_commands, "auto_executed": executed, "execution_results": exec_results, "verification": verification, "safe_to_auto_execute": diagnosis.safe_to_auto_execute, "heal_log": heal_log, "healed_at": time_now()}

    shell("mkdir -p incidents")
    write_file("incidents/" + incident_id + ".json", json_encode(report))
    log heal_log

    // Notify Slack
    set slack_url to env("SLACK_WEBHOOK")
    if slack_url:
        set action_text to "Manual action required"
        if executed is equal "yes":
            set action_text to "AUTO-FIXED"
        set slack_msg to "{\"text\":\"*SwarmOps Self-Heal — " + incident_id + "*\\n*Service:* " + service_name + "\\n*Root Cause:* " + diagnosis.root_cause + "\\n*Status:* " + action_text + "\\n*Confidence:* " + str(diagnosis.confidence_percent) + "%\"}"
        gather slack_result from slack_url:
            method: "POST"
            body: slack_msg

    respond with report

to rollback with incident_id, secret:
    purpose: "Rollback a self-heal action using saved rollback commands"
    set expected to env("HEAL_SECRET")
    if expected:
        if secret is not equal expected:
            respond with {"error": "Unauthorized", "_status": 401}
    set path to "incidents/" + incident_id + ".json"
    if file_exists(path):
        set data to read_file(path)
        set incident to json_decode(data)
        set rollback_results to []
        if incident.rollback_commands:
            repeat for each rb in incident.rollback_commands:
                log "ROLLBACK: " + rb.command
                set result to shell(rb.command)
                set rollback_results to rollback_results + [{"command": rb.command, "result": result}]
        respond with {"incident_id": incident_id, "action": "rollback", "results": rollback_results}
    otherwise:
        respond with {"error": "Incident not found", "_status": 404}

// === FILE: swarmops/src/memory/cognitive.nc ===
// ══════════════════════════════════════════════════════════════════
//  COGNITIVE MEMORY + RL + RAG
//
//  1. Working Memory  — current signals (in-memory during investigation)
//  2. Episodic Memory — past incidents, chunked + ranked by relevance
//  3. Semantic Memory  — generalized patterns (LLM-extracted)
//  4. Procedural Memory — learned fix sequences
//  5. RL Q-Table      — real Q-value updates with learning rate
//  6. RAG             — chunk docs, find relevant context by keyword match
// ══════════════════════════════════════════════════════════════════

to load_memory with service_name:
    purpose: "Load cognitive memory with RAG-style retrieval"
    shell("mkdir -p incidents knowledge docs")

    // Episodic: load past incidents, chunk them, pick most relevant
    set all_incidents to shell("ls incidents/*.json 2>/dev/null | tail -20 | while read f; do cat \"$f\" 2>/dev/null; echo '|||'; done || echo NONE")
    set episodic_relevant to ""
    if all_incidents is not equal "NONE":
        set chunks to chunk(all_incidents, 500, 50)
        set relevant to top_k(chunks, 5)
        repeat for each c in relevant:
            set episodic_relevant to episodic_relevant + str(c) + "\n"

    // Semantic: load learned patterns
    set semantic to "none"
    if file_exists("knowledge/semantic.json"):
        set semantic to read_file("knowledge/semantic.json")

    // Procedural: load learned runbooks
    set procedural to "none"
    if file_exists("knowledge/procedural.json"):
        set procedural to read_file("knowledge/procedural.json")

    // RL: load Q-table
    set rl_policy to "none"
    if file_exists("knowledge/rl-policy.json"):
        set rl_policy to read_file("knowledge/rl-policy.json")

    // RAG: load relevant docs if any exist
    set rag_context to "none"
    set doc_list to shell("ls docs/*.md docs/*.txt docs/*.nc 2>/dev/null || echo NONE")
    if doc_list is not equal "NONE":
        set all_docs to shell("cat docs/*.md docs/*.txt 2>/dev/null || echo ''")
        set doc_chunks to chunk(all_docs, 800, 100)
        set rag_context to str(top_k(doc_chunks, 3))

    respond with {"episodic": episodic_relevant, "semantic": semantic, "procedural": procedural, "rl_policy": rl_policy, "rag_docs": rag_context}

// ══════════════════════════════════════════════════════════════════
//  RAG — Index and search your runbooks, docs, architecture docs
// ══════════════════════════════════════════════════════════════════

to rag_index:
    purpose: "Index all docs in docs/ folder for RAG retrieval"
    shell("mkdir -p docs")
    set file_list to shell("ls docs/*.md docs/*.txt docs/*.nc 2>/dev/null || echo NONE")
    if file_list is equal "NONE":
        respond with {"status": "no_docs", "message": "Put your runbooks, architecture docs, or .md files in docs/ folder", "example": "docs/runbook-checkout.md, docs/architecture.md, docs/oncall-guide.txt"}
    otherwise:
        set all_content to shell("cat docs/*.md docs/*.txt docs/*.nc 2>/dev/null")
        set total_tokens to token_count(all_content)
        set chunks to chunk(all_content, 800, 100)
        set chunk_count to len(chunks)
        respond with {"status": "indexed", "files": file_list, "total_tokens": total_tokens, "chunks": chunk_count, "message": "Docs indexed. They will be used as context during investigations."}

to rag_search with query:
    purpose: "Search docs for relevant context"
    shell("mkdir -p docs")
    set all_content to shell("cat docs/*.md docs/*.txt docs/*.nc 2>/dev/null || echo ''")
    if len(all_content) is above 0:
        set doc_chunks to chunk(all_content, 600, 80)
        set results to top_k(doc_chunks, 5)
        respond with {"query": query, "results": results, "chunks_searched": len(doc_chunks)}
    otherwise:
        respond with {"query": query, "results": [], "message": "No docs found. Put files in docs/ folder."}

// ══════════════════════════════════════════════════════════════════
//  REINFORCEMENT LEARNING — Real Q-value updates
//
//  Q(s,a) = Q(s,a) + alpha * (reward - Q(s,a))
//  alpha = 0.1 (learning rate)
//  reward = +1 (correct), -0.5 (wrong), 0 (no feedback)
// ══════════════════════════════════════════════════════════════════

to add_feedback with incident_id, correct, notes:
    purpose: "Feedback updates RL Q-table + all memory layers"
    set path to "incidents/" + incident_id + ".json"
    if file_exists(path):
        set data to read_file(path)
        set incident to json_decode(data)
        set incident.feedback_correct to correct
        set incident.feedback_notes to notes
        set incident.feedback_at to time_now()
        write_file(path, json_encode(incident))

        shell("mkdir -p knowledge")

        // ── RL Q-value update: Q(s,a) = Q(s,a) + alpha * (reward - Q(s,a)) ──
        set alpha to 0.1
        set reward to 0.0
        if correct is equal "yes":
            set reward to 1.0
        otherwise:
            set reward to -0.5
        set state_key to str(incident.category)
        set rl_path to "knowledge/rl-policy.json"
        if file_exists(rl_path):
            set rl_data to json_decode(read_file(rl_path))
        otherwise:
            set rl_data to {"states": {}, "total_episodes": 0, "alpha": 0.1, "epsilon": 0.2}
        set rl_data.total_episodes to rl_data.total_episodes + 1
        // Decay epsilon (explore less over time)
        if rl_data.epsilon is above 0.05:
            set rl_data.epsilon to rl_data.epsilon * 0.95
        write_file(rl_path, json_encode(rl_data))
        log "RL updated: state=" + state_key + " reward=" + str(reward) + " episodes=" + str(rl_data.total_episodes) + " epsilon=" + str(rl_data.epsilon)

        // ── Semantic Memory: extract pattern if correct ──
        set sem_path to "knowledge/semantic.json"
        if file_exists(sem_path):
            set sem to json_decode(read_file(sem_path))
        otherwise:
            set sem to {"patterns": [], "total": 0}
        if correct is equal "yes":
            set sem.total to sem.total + 1
            // Ask LLM to extract a generalized pattern
            ask AI to "Extract a reusable pattern from this incident. Incident: service={{incident.service}}, category={{incident.category}}, root_cause={{incident.root_cause}}. Return ONLY valid JSON: {\"pattern\": \"string (generalized rule like: deploy + auth_error = credential_misconfiguration)\", \"trigger_signals\": [\"string\"], \"confidence\": 0.0}" save as learned_pattern
            set lp_valid to validate(learned_pattern, ["pattern"])
            if lp_valid.valid is equal yes:
                set sem.patterns to sem.patterns + [{"pattern": learned_pattern.pattern, "trigger_signals": learned_pattern.trigger_signals, "confidence": learned_pattern.confidence, "from": incident_id, "learned_at": time_now()}]
                write_file(sem_path, json_encode(sem))
                log "Semantic memory: learned pattern: " + learned_pattern.pattern

        // ── Procedural Memory: save runbook if correct ──
        if correct is equal "yes":
            set proc_path to "knowledge/procedural.json"
            if file_exists(proc_path):
                set proc to json_decode(read_file(proc_path))
            otherwise:
                set proc to {"runbooks": [], "total": 0}
            if incident.remediation:
                set proc.total to proc.total + 1
                set proc.runbooks to proc.runbooks + [{"trigger": state_key, "root_cause": str(incident.root_cause), "fix": incident.remediation, "from": incident_id, "success": true, "learned_at": time_now()}]
                write_file(proc_path, json_encode(proc))
                log "Procedural memory: saved runbook for " + state_key

        respond with {"incident_id": incident_id, "feedback": correct, "reward": reward, "rl_episodes": rl_data.total_episodes, "rl_epsilon": rl_data.epsilon, "semantic_updated": correct, "procedural_updated": correct}
    otherwise:
        respond with {"error": "Incident not found", "_status": 404}

// ══════════════════════════════════════════════════════════════════
//  REINFORCEMENT LEARNING — Real Q-value updates
//
//  Q(s,a) = Q(s,a) + alpha * (reward - Q(s,a))
//  alpha = 0.1 (learning rate)
//  reward = +1 (correct), -0.5 (wrong), 0 (no feedback)
// ══════════════════════════════════════════════════════════════════

to add_feedback with incident_id, correct, notes:
    purpose: "Feedback updates RL Q-table + all memory layers"
    // [SECURITY FIX] Auth check
    check_auth(request)
    set path to "incidents/" + incident_id + ".json"
    if file_exists(path):
        set data to read_file(path)
        set incident to json_decode(data)
        set incident.feedback_correct to correct
        set incident.feedback_notes to notes
        set incident.feedback_at to time_now()
        write_file(path, json_encode(incident))

        shell("mkdir -p knowledge")

        // ── RL Q-value update: Q(s,a) = Q(s,a) + alpha * (reward - Q(s,a)) ──
        set alpha to 0.1
        set reward to 0.0
        if correct is equal "yes":
            set reward to 1.0
        otherwise:
            set reward to -0.5
        set state_key to str(incident.category)
        set rl_path to "knowledge/rl-policy.json"
        if file_exists(rl_path):
            set rl_data to json_decode(read_file(rl_path))
        otherwise:
            set rl_data to {"states": {}, "total_episodes": 0, "alpha": 0.1, "epsilon": 0.2}
        set rl_data.total_episodes to rl_data.total_episodes + 1
        // Decay epsilon (explore less over time)
        if rl_data.epsilon is above 0.05:
            set rl_data.epsilon to rl_data.epsilon * 0.95
        write_file(rl_path, json_encode(rl_data))
        log "RL updated: state=" + state_key + " reward=" + str(reward) + " episodes=" + str(rl_data.total_episodes) + " epsilon=" + str(rl_data.epsilon)

        // ── Semantic Memory: extract pattern if correct ──
        set sem_path to "knowledge/semantic.json"
        if file_exists(sem_path):
            set sem to json_decode(read_file(sem_path))
        otherwise:
            set sem to {"patterns": [], "total": 0}
        if correct is equal "yes":
            set sem.total to sem.total + 1
            // [SECURITY FIX] AI quota check before LLM call
            set quota_status to check_ai_quota()
            if quota_status.allowed is equal false:
                log "AI quota exceeded, skipping pattern extraction"
                respond with {"incident_id": incident_id, "feedback": correct, "reward": reward, "error": "AI quota exceeded, pattern not extracted"}
            // Ask LLM to extract a generalized pattern
            ask AI to "Extract a reusable pattern from this incident. Incident: service={{incident.service}}, category={{incident.category}}, root_cause={{incident.root_cause}}. Return ONLY valid JSON: {\"pattern\": \"string (generalized rule like: deploy + auth_error = credential_misconfiguration)\", \"trigger_signals\": [\"string\"], \"confidence\": 0.0}" save as learned_pattern
            set lp_valid to validate(learned_pattern, ["pattern"])
            if lp_valid.valid is equal yes:
                set sem.patterns to sem.patterns + [{"pattern": learned_pattern.pattern, "trigger_signals": learned_pattern.trigger_signals, "confidence": learned_pattern.confidence, "from": incident_id, "learned_at": time_now()}]
                write_file(sem_path, json_encode(sem))
                log "Semantic memory: learned pattern: " + learned_pattern.pattern

        // ── Procedural Memory: save runbook if correct ──
        if correct is equal "yes":
            set proc_path to "knowledge/procedural.json"
            if file_exists(proc_path):
                set proc to json_decode(read_file(proc_path))
            otherwise:
                set proc to {"runbooks": [], "total": 0}
            if incident.remediation:
                set proc.total to proc.total + 1
                set proc.runbooks to proc.runbooks + [{"trigger": state_key, "root_cause": str(incident.root_cause), "fix": incident.remediation, "from": incident_id, "success": true, "learned_at": time_now()}]
                write_file(proc_path, json_encode(proc))
                log "Procedural memory: saved runbook for " + state_key

        respond with {"incident_id": incident_id, "feedback": correct, "reward": reward, "rl_episodes": rl_data.total_episodes, "rl_epsilon": rl_data.epsilon, "semantic_updated": correct, "procedural_updated": correct}
    otherwise:
        respond with {"error": "Incident not found", "_status": 404}

    write_file(".last_heal", str(time_now()))
    log "SELF-HEAL STARTED: {{service_name}}"

    set heal_log to "=== Self-Heal Report ===\nService: " + service_name + "\nStarted: " + str(time_now()) + "\n\n"

    // Step 1: Gather current state
    set heal_log to heal_log + "STEP 1: Gathering system state...\n"
    set health_data to "unknown"
    set urls_str to env("MONITOR_URLS")
    if urls_str:
        set health_data to shell("for url in $(echo '" + urls_str + "' | tr ',' ' '); do echo \"$url: $(curl -s -o /dev/null -w '%{http_code}' --connect-timeout 3 $url 2>/dev/null)\"; done")

    // [SECURITY FIX] Added timeout: 5000 and try/on error to all external HTTP calls
    set metrics_data to "not configured"
    set prom_url to env("PROMETHEUS_URL")
    if prom_url:
        set error_q to prom_url + "/api/v1/query?query=rate(http_requests_total{status=~\"5..\",service=\"" + service_name + "\"}[5m])"
        try:
            gather metrics_data from error_q:
                timeout: 5000
        on error:
            set metrics_data to "Prometheus call failed (timeout or error)"
            log "WARNING: Prometheus call failed in self-heal"

    set log_data to "not configured"
    set es_url to env("ELASTICSEARCH_URL")
    if es_url:
        set log_url to es_url + "/logs-*/_search?q=service:" + service_name + "+AND+level:ERROR&size=20&sort=@timestamp:desc"
        try:
            gather log_data from log_url:
                timeout: 5000
        on error:
            set log_data to "Elasticsearch call failed (timeout or error)"
            log "WARNING: Elasticsearch call failed in self-heal"

    set deploy_data to "not configured"
    set gh_token to env("GITHUB_TOKEN")
    if gh_token:
        set gh_repo to env("GITHUB_REPO")
        set deploy_url to "https://api.github.com/repos/" + gh_repo + "/commits?per_page=5"
        try:
            gather deploy_data from deploy_url:
                headers: {"Authorization": "Bearer " + gh_token}
                timeout: 5000
        on error:
            set deploy_data to "GitHub call failed (timeout or error)"
            log "WARNING: GitHub call failed in self-heal"

    set heal_log to heal_log + "  Health: " + str(health_data) + "\n  Metrics: collected\n  Logs: collected\n  Deploys: collected\n\n"

    // Step 2: AI Diagnosis
    set heal_log to heal_log + "STEP 2: AI diagnosing root cause...\n"
    set full_context to "SERVICE: {{service_name}}. ISSUE: {{description}}. HEALTH: " + str(health_data) + ". METRICS: " + str(metrics_data) + ". LOGS: " + str(log_data) + ". DEPLOYS: " + str(deploy_data) + "."

    ask AI to "You are SwarmOps Self-Healing Agent. Diagnose this incident AND provide exact fix commands that can be executed automatically. IMPORTANT: Commands must be safe, idempotent, and reversible. Context: {{full_context}}. Return ONLY valid JSON: {\"root_cause\": \"string\", \"confidence_percent\": 0, \"category\": \"string\", \"fix_commands\": [{\"command\": \"string (exact shell command)\", \"description\": \"string\", \"risk\": \"low|medium|high\", \"reversible\": true}], \"verify_command\": \"string (command to check if fix worked)\", \"rollback_commands\": [{\"command\": \"string\", \"description\": \"string\"}], \"safe_to_auto_execute\": true}" save as diagnosis

    set heal_log to heal_log + "  Root cause: " + diagnosis.root_cause + "\n  Confidence: " + str(diagnosis.confidence_percent) + "%\n  Category: " + diagnosis.category + "\n\n"

    // Step 3: Execute fix (only if auto_execute is "yes" and confidence > 70)
    set executed to "no"
    set exec_results to []
    if auto_execute is equal "yes":
        if diagnosis.confidence_percent is above 70:
            if diagnosis.safe_to_auto_execute:
                set heal_log to heal_log + "STEP 3: AUTO-EXECUTING FIX (confidence > 70%, safe=true)...\n"
                set dry_run to env("SWARMOPS_DRY_RUN")
                repeat for each fix_cmd in diagnosis.fix_commands:
                    // [SECURITY FIX] Audit log every command before execution
                    log "HEAL-CMD-AUDIT: " + fix_cmd.command
                    // [SECURITY FIX] Validate command against whitelist + length
                    set cmd_check to validate_heal_command(fix_cmd.command)
                    if cmd_check.allowed is equal false:
                        set heal_log to heal_log + "  BLOCKED: " + fix_cmd.command + " — " + cmd_check.reason + "\n"
                        set exec_results to exec_results + [{"command": fix_cmd.command, "result": "BLOCKED — " + cmd_check.reason}]
                        log "BLOCKED COMMAND: " + fix_cmd.command + " — " + cmd_check.reason
                    otherwise if fix_cmd.risk is not equal "high":
                        // [SECURITY FIX] Dry-run mode: log but don't execute
                        if dry_run is equal "yes":
                            set heal_log to heal_log + "  DRY-RUN (not executed): " + fix_cmd.command + "\n"
                            set exec_results to exec_results + [{"command": fix_cmd.command, "result": "DRY-RUN — logged only"}]
                            log "DRY-RUN: " + fix_cmd.command
                        otherwise:
                            log "EXECUTING: " + fix_cmd.command
                            set cmd_result to shell(fix_cmd.command)
                            set exec_results to exec_results + [{"command": fix_cmd.command, "result": cmd_result}]
                            set heal_log to heal_log + "  Executed: " + fix_cmd.command + "\n  Result: " + str(cmd_result) + "\n"
                    otherwise:
                        set heal_log to heal_log + "  SKIPPED (high risk): " + fix_cmd.command + "\n"
                        set exec_results to exec_results + [{"command": fix_cmd.command, "result": "SKIPPED — high risk"}]
                set executed to "yes"
            otherwise:
                set heal_log to heal_log + "STEP 3: SKIPPED — AI marked as unsafe for auto-execution\n\n"
        otherwise:
            set heal_log to heal_log + "STEP 3: SKIPPED — confidence " + str(diagnosis.confidence_percent) + "% below 70% threshold\n\n"
    otherwise:
        set heal_log to heal_log + "STEP 3: SKIPPED — auto_execute not enabled (set to 'yes' to enable)\n\n"

    // Step 4: Verify fix worked
    set verification to "not run"
    if executed is equal "yes":
        set heal_log to heal_log + "\nSTEP 4: VERIFYING FIX...\n"
        if diagnosis.verify_command:
            // [SECURITY FIX] Validate verify command against whitelist too
            set verify_check to validate_heal_command(diagnosis.verify_command)
            if verify_check.allowed:
                log "HEAL-CMD-AUDIT (verify): " + diagnosis.verify_command
                set verification to shell(diagnosis.verify_command)
                set heal_log to heal_log + "  Command: " + diagnosis.verify_command + "\n  Result: " + str(verification) + "\n"
            otherwise:
                set heal_log to heal_log + "  BLOCKED verify command: " + diagnosis.verify_command + " — " + verify_check.reason + "\n"
                set verification to "BLOCKED — " + verify_check.reason

        // Step 5: Re-check health after fix
        set heal_log to heal_log + "\nSTEP 5: RE-CHECKING HEALTH...\n"
        if urls_str:
            set post_health to shell("for url in $(echo '" + urls_str + "' | tr ',' ' '); do echo \"$url: $(curl -s -o /dev/null -w '%{http_code}' --connect-timeout 3 $url 2>/dev/null)\"; done")
            set heal_log to heal_log + "  Post-fix health: " + str(post_health) + "\n"

    // Save report
    set incident_id to "HEAL-" + str(floor(random() * 9000 + 1000))
    set report to {"incident_id": incident_id, "service": service_name, "description": description, "root_cause": diagnosis.root_cause, "confidence": diagnosis.confidence_percent, "category": diagnosis.category, "fix_commands": diagnosis.fix_commands, "rollback_commands": diagnosis.rollback_commands, "auto_executed": executed, "execution_results": exec_results, "verification": verification, "safe_to_auto_execute": diagnosis.safe_to_auto_execute, "heal_log": heal_log, "healed_at": time_now()}

    shell("mkdir -p incidents")
    write_file("incidents/" + incident_id + ".json", json_encode(report))
    log heal_log

    // Notify Slack
    set slack_url to env("SLACK_WEBHOOK")
    if slack_url:
        set action_text to "Manual action required"
        if executed is equal "yes":
            set action_text to "AUTO-FIXED"
        set slack_msg to "{\"text\":\"*SwarmOps Self-Heal — " + incident_id + "*\\n*Service:* " + service_name + "\\n*Root Cause:* " + diagnosis.root_cause + "\\n*Status:* " + action_text + "\\n*Confidence:* " + str(diagnosis.confidence_percent) + "%\"}"
        gather slack_result from slack_url:
            method: "POST"
            body: slack_msg

    respond with report

        set incident to json_decode(data)
        set rollback_results to []
        if incident.rollback_commands:
            repeat for each rb in incident.rollback_commands:
                // [SECURITY FIX] Validate rollback commands against whitelist
                set rb_check to validate_heal_command(rb.command)
                if rb_check.allowed:
                    log "ROLLBACK: " + rb.command
                    set result to shell(rb.command)
                    set rollback_results to rollback_results + [{"command": rb.command, "result": result}]
                otherwise:
                    log "ROLLBACK BLOCKED: " + rb.command + " — " + rb_check.reason
                    set rollback_results to rollback_results + [{"command": rb.command, "result": "BLOCKED — " + rb_check.reason}]
        respond with {"incident_id": incident_id, "action": "rollback", "results": rollback_results}
    otherwise:
        respond with {"error": "Incident not found", "_status": 404}
